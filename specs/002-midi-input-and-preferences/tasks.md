## Implementation Strategy

### MVP First (US1 only)

1. Phase 1 Setup (T001–T007).
2. Phase 2 Foundational (T008–T020). The native module is the biggest single investment; build + test on one platform first (usually iOS in this project) to validate the end-to-end pipeline before duplicating on Android.
3. Phase 3 US1 tests (T021–T027), confirm all fail.
4. Phase 3 US1 implementation (T028–T035), run tests → green.
5. Manual smoke T043 on iOS with a physical controller.
6. **Ship**: MVP is a working MIDI activity readout in the Edit header.

### Incremental Delivery

1. MVP as above (US1).
2. Add US2 (T036–T039): toggle behavior works end-to-end; persistence not yet proven.
3. Add US3 (T040): persistence locked in by regression test.
4. Polish (T041–T045): typecheck, full jest, iOS + Android manual smoke, keyboard nav.
